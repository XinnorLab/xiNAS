# Filesystem Parameters ā€” Health Check Tunables

XFS mount options and mkfs parameters verified by the xiNAS health check engine.
These settings are applied automatically by the `raid_fs` and `nvme_namespace` Ansible roles.

## XFS Mount Options

All data filesystems are mounted under `/mnt/` with options generated by the
`nvme_namespace` role (`generate_raid_config.yml`). The health check reads
`/proc/mounts` and verifies each option is present.

| Option | Recommended | Why it matters | How to check | Remediation |
|--------|-------------|----------------|--------------|-------------|
| `noatime` | present | Prevents metadata writes on every read ā€” critical for NFS serving where clients generate millions of reads | `grep /mnt /proc/mounts` | Add `noatime` to mount options in `/etc/fstab` or systemd mount unit |
| `nodiratime` | present | Disables atime updates on directory access ā€” reduces metadata I/O for directory-heavy NFS workloads (ls, find) | `grep /mnt /proc/mounts` | Add `nodiratime` to mount options |
| `logdev=<device>` | present (e.g., `/dev/xi_log`) | Separates journal writes from data I/O onto a dedicated RAID-10 log array ā€” eliminates journal contention on the data disks | `grep /mnt /proc/mounts` | Requires `mkfs.xfs -l logdev=...` at filesystem creation time; cannot be changed post-creation without reformatting |
| `logbsize=256k` | `256k` | Increases the in-memory log buffer from the default 32 KB ā€” reduces log flushes for sequential write workloads | `grep /mnt /proc/mounts` | Add `logbsize=256k` to mount options |
| `allocsize=131072k` | `131072k` (128 MB) | Sets the preallocation hint for buffered writes ā€” reduces fragmentation for large file writes typical in NFS/AI workloads | `grep /mnt /proc/mounts` | Add `allocsize=131072k` to mount options |
| `largeio` | present | Causes `stat()` and `pathconf()` to report the stripe width as the preferred I/O size ā€” NFS clients use this to optimize transfer sizes | `grep /mnt /proc/mounts` | Add `largeio` to mount options |
| `inode64` | present | Allows inode allocation across the entire filesystem ā€” without this, inodes are constrained to the first 1 TB, causing allocation failures on large volumes | `grep /mnt /proc/mounts` | Add `inode64` to mount options (default on modern kernels, but verify) |
| `swalloc` | present | Forces stripe-aligned allocation ā€” ensures data blocks align with the RAID stripe geometry for optimal throughput | `grep /mnt /proc/mounts` | Add `swalloc` to mount options |

## XFS Stripe Alignment (mkfs parameters)

Set at filesystem creation time via `mkfs.xfs -d su=<strip_size>k,sw=<stripe_width>`.
Cannot be changed without reformatting.

| Parameter | Recommended | Why it matters | How to check |
|-----------|-------------|----------------|--------------|
| `sunit` (stripe unit) | Non-zero, matching RAID strip size | Aligns allocation unit with RAID strip size ā€” misalignment causes read-modify-write penalties on every I/O | `xfs_info /mnt/<mountpoint>` ā€” look for `sunit=N` (in 512-byte blocks) |
| `swidth` (stripe width) | Non-zero, matching RAID data disks x strip size | Aligns full-stripe writes ā€” enables the RAID controller to write entire stripes without read-modify-write | `xfs_info /mnt/<mountpoint>` ā€” look for `swidth=N` (in 512-byte blocks) |

**Typical values:**
- Data array (RAID 5, 128 KB strip): `sunit=256` (128 KB / 512), `swidth=256 * (N-1)` where N = drive count
- Log array (RAID 10, 16 KB strip): `sunit=32` (16 KB / 512), `swidth=32 * (N/2)`

## XFS Quota

| Parameter | Recommended | Why it matters |
|-----------|-------------|----------------|
| `uquota` | present | Enables user quota tracking ā€” required for per-user space accounting on NFS exports |

## Health Check Integration

- **Section**: `filesystem`
- **Check names**: `mount_opts` (existing, checks noatime), `xfs_mount_opts` (new, checks all options above), `xfs_stripe_align` (new)
- **Profile coverage**: `xfs_mount_opts` in standard+deep; `xfs_stripe_align` in deep only
- **Severity**: Missing options ā†’ WARN; missing stripe alignment ā†’ WARN
- **Source files**: `collection/roles/nvme_namespace/tasks/generate_raid_config.yml`, `collection/roles/raid_fs/tasks/create_fs.yml`
